# 🤝 Contributing Guide

Contributions to Mind Library are welcome!

## 🚀 How to Contribute

### 1. Report Issues

If you find a bug or have a feature request:
1. Check if a similar issue already exists
2. Create a new Issue
3. Describe the problem and reproduction steps in detail

### 2. Submit Code

```bash
# 1. Fork the project
# 2. Clone your fork
git clone https://github.com/epantrip/mind-lib.git
cd mind-lib

# 3. Create a feature branch
git checkout -b feature/your-feature

# 4. Develop and test
# ... make changes ...

# 5. Commit
git add .
git commit -m "Add: your new feature"

# 6. Push
git push origin feature/your-feature

# 7. Create a Pull Request
```

### 3. Write Documentation

- Fix documentation errors
- Fill in missing content
- Translate to other languages

## 📋 Code Style

### Python
- Follow PEP 8
- Use 4-space indentation
- Max line length: 100 characters
- Type annotations are optional but recommended

### Commit Messages

```
Type: Short description

Detailed explanation (optional)

Fixes: #Issue number
```

**Type prefixes:**
- `Add:` New feature
- `Fix:` Bug fix
- `Update:` Update existing feature
- `Docs:` Documentation change
- `Refactor:` Code refactoring
- `Test:` Test-related

## 🧪 Testing Standards

- New features must have test coverage
- Ensure all tests pass before submitting
- Keep test code clean and readable

## 💬 Communication

- **GitHub Issues:** Primary communication channel
- Design discussions are welcome in Issues

---

Thank you for contributing! 🎃
